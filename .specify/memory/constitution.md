# perf-toolkit — 開發憲章

## 開發原則

### 工具不做決定
- 系統提供數據與參考，最終判斷永遠是主管的
- 評分結果僅作為參考依據，不取代人為判斷
- 等第分配由主管綜合考量後決定

### 輕量優先
- 不過度工程化，10 人團隊不需要 enterprise 方案
- 使用 JSON/CSV 而非資料庫，零部署成本
- 資料可直接版控，fork 後不用設定環境就能用

### 漸進式採用
- 可以只用其中一個模組，不需要全套導入
- 各模組之間低耦合，獨立可用
- 從文件開始，逐步加入自動化

### 隱私敏感
- 預設脫敏，敏感資料不進版控
- 範例資料一律使用假名
- 公佈版報告不含排名與負面備註

## 技術限制

### 技術棧
- **前端**：React 或 Vue（待定）
- **資料**：JSON / CSV，不依賴資料庫
- **報告**：HTML 模板 + CSS print media（A4 列印）
- **腳本**：Python（考勤分析、Excel 處理）
- **部署**：GitHub Pages（靜態）或本地運行

### 最少依賴原則
- 能用原生 API 就不引入套件
- Python 腳本盡量只用標準庫 + pandas
- 前端框架選定後統一，不混用

### 瀏覽器支援
- 現代瀏覽器（Chrome、Firefox、Safari、Edge）
- 報告 HTML 需支援列印為 A4 PDF

## 程式碼品質

### 風格
- 清晰易讀，變數命名有意義
- 一致的格式（使用 linter/formatter）
- 關注點分離：資料處理、UI、報告產出各自獨立

### 測試
- 考勤分析腳本需有單元測試
- 報告產生器用範例資料驗證輸出
- 多瀏覽器測試列印效果
